Building Knuth-Morris-Pratt (KMP) Pattern Matching

= Context

The KMP algorithm describes a method for finding a length-m string in a length-n string in O(m+n) time.

There are plenty of online resources describing the algorithm’s implementation. I couldn’t find many satisfactory explanations as to why it works, so I decided to write up some notes myself.

This post documents an effort to construct the KMP Algorithm from the ground up. It’ll focus on the intuitions behind the process; to get familiar with the mechanics of the algorithm, I’d recommend watching Tushar Roy’s excellent two-part video on the topic https://youtu.be/GTJr8OvyEVQ[here] and https://youtu.be/KG44VoDtsAA[here].

Given yeemyeelyeefyeet, find yeet.

Again, pretty simple; go through, find a y, keep matching. If our match fails, just keep on going through the string looking for more ys as before.

This case illustrates our first issue any “one-pass” algorithm must encounter:

  1. Sometimes, we’ll get a mismatch only at the end of a pattern.

This isn’t always a problem. In our example, we scan for the letter y. If we find y, then we have found a “candidate string” for yeet (ie. the y could potentially be the start of the word yeet). To determine if our candidate string is valid, we continue matching until we either succeed finding yeet or find a mismatch (eg. yeel).

Take a look at our numbered candidate strings below:

However, there are some “problem strings” for which this approach doesn’t work. Consider the following example:

This brings us to Issue 2:

  1. Sometimes, candidate strings can overlap.

What causes a candidate string to overlap? We see that it depends entirely on the string that we’re searching for; possible matches for yash will never overlap with each other. Let’s try to isolate exactly what properties of the search pattern cause this to happen.

For two candidate strings to overlap as above, the pattern must overlap with itself (at least partially). yash, for example, can never overlap with itself (it starts with a y, which is only present once in the string). To state more formally, two candiate strings can “overlap” if some part of the pattern is also a prefix to the pattern.
